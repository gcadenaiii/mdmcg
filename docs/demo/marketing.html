Powerful Features

Everything you need for professional motion tracking and sensor data visualization

🎨

Real-Time 3D Visualization

Beautiful Three.js-powered 3D visualization with live sensor orientation, acceleration vectors, and motion trails. See your sensor move in real-time at 20Hz.

📡

Multiple Transport Options

Connect via USB Serial, Bluetooth LE, or WiFi (ESP32). Switch between protocols without changing your application code.

🔧

Web-Based Configuration

Intuitive web interface for sensor calibration, configuration management, and real-time monitoring. No command-line required.

📊

Data Logging & Analysis

Automatic HDF5 data logging with timestamp precision. Export and analyze your motion data with industry-standard tools.

🧪

Mock Mode for Development

Develop and test without hardware. Mock sensor generates realistic motion data with configurable noise and movement patterns.

🔐

VPN Remote Access

Secure WireGuard VPN support for remote sensor access. Monitor and control sensors from anywhere in the world.

Supported Hardware Profiles

Flexible architecture supports multiple sensor configurations

Profile Hardware Connection Use Case
mock No hardware Simulated Development & testing without hardware
bno055 BNO055 sensor I²C (Raspberry Pi GPIO) Direct sensor connection to Pi
arduino Arduino + BNO055 USB Serial Wired Arduino connection via USB
arduino_ble Arduino Nano 33 BLE + BNO055 Bluetooth Low Energy Wireless BLE streaming
esp32_wifi ESP32 + BNO055 WiFi (WebSocket) Network-based wireless streaming

Quick Start

Get up and running in under 5 minutes

# Clone the repository
git clone https://github.com/gcadenaiii/sensors_lab.git
cd sensors_lab

# Create virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
pip install -e .

# Start web interface (mock mode - no hardware needed)
sensors-track --mode web --profile mock

# Open browser to http://localhost:8000
